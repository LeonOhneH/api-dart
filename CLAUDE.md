## Project Overview

Dart REST API that scrapes fussball.de (German football/soccer website) and serves structured JSON data. Built with `shelf` web framework and `Isar` embedded database. Deployed via GitHub Actions CI/CD to a server managed by PM2.

## Common Commands

```bash
# Install dependencies
dart pub get

# Run server (port 8080 by default, configurable via PORT env var)
dart run bin/server.dart

# Run all tests (MUST use -j 1 to avoid database conflicts)
dart test -j 1

# Run a single test file
dart test -j 1 test/unit/html/games_test.dart

# Static analysis
dart analyze

# Check for auto-fixable issues (CI fails if fixes are available)
dart fix --dry-run

# Apply fixes
dart fix --apply

# Regenerate Isar entity code (after modifying entities)
dart run build_runner build

# Compile to native executable
dart compile exe bin/server.dart -o bin/server
```

## Architecture

**Request flow:** `bin/server.dart` (shelf Router) → middleware (auth token check, rate limiting, JSON headers, error handling) → `lib/controller/` → `lib/html/` (HTML parsers) → response DTOs → JSON

**Key layers:**
- **`lib/controller/`** - Route handlers. `ApiController` handles all data endpoints; `AuthController` handles user registration.
- **`lib/html/`** - HTML parsing logic. Each parser takes raw HTML from fussball.de and extracts structured data. `Games` parses match lists, `Club` parses team listings, `TableResult` parses league tables, `Font` handles score obfuscation decoding.
- **`lib/crawler/http_client_bridge.dart`** - HTTP client abstraction for fetching fussball.de pages. Uses `HttpClientBridgeInterface` for testability.
- **`lib/dto/`** - Data transfer objects for API responses and parsed data.
- **`lib/entities/`** - Isar database entities (`User`, `Font`, `RateLimit`). Files ending in `.g.dart` are auto-generated by `build_runner` — do not edit manually.
- **`lib/database.dart`** - Isar singleton, query helpers for users/fonts/rate-limits.
- **`lib/middleware.dart`** - Auth token validation (via `x-auth-token` header), rate limiting (30 req/min per user), JSON content-type, global error handler.

**Font obfuscation:** fussball.de encodes match scores using custom WOFF fonts. The `Font` class downloads these fonts, converts them to TTX via the `ttx` command-line tool (from `fonttools`), and parses the cmap to build a character mapping. `FontProxy` caches decoded fonts in Isar. The `fonttools` system package is required for this to work.

**`bin/` scripts:** `server.dart` (main API server), `backup.dart` (export users to JSON), `dump.dart` (database dump), `clear.dart` (clear database).

## Testing

Tests are organized into `test/unit/`, `test/integration/`, and `test/api/`. Unit tests for HTML parsers use inline HTML fixtures. The font_proxy integration test is skipped in CI (requires `fonttools`). Always run tests with `-j 1`.
